##  Assumptions Made

1. The project is frontend-only and does not require backend integration.
2. Login, Sign Up, and Contact functionalities are UI-based and not connected to any server.
3. Placeholder text (Lorem Ipsum) was used where actual content was not provided in the design.
4. Image assets are stored inside the `/assets` folder.
5. Dropdown menu is implemented using CSS hover behavior for simplicity.
6. FAQ accordion animation uses a fixed `max-height` for smooth transitions.
7. Responsive behavior is designed primarily for desktop, tablet (â‰¤992px), and mobile (â‰¤768px) breakpoints.
8. The website is intended to be deployed as a static site (e.g., GitHub Pages).
##  Known Issues

1. Dropdown navigation is hover-based and may require JavaScript enhancement for better mobile support.
2. FAQ animation may require adjustment if answers become significantly longer.
3. No form validation is implemented for the Subscribe or Contact section.
4. Accessibility features (ARIA roles, keyboard navigation, screen reader support) can be improved.
5. No backend authentication system is implemented.
6. Some layout spacing may slightly vary across different browsers.
7. Horizontal scroll in the Reviews section depends on browser behavior.
